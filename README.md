# 物理信息驱动的少样本跨站点光伏功率生成框架
## (Physics-Informed Few-Shot PV Generation Framework)

### 1. 设计理念 (Design Philosophy)
本模型旨在解决光伏预测中的**“少样本跨站点泛化”**难题。核心思想是将光伏功率生成分解为两个独立变量：
*   **天气背景 (Weather/Transient)**：由 NWP（气象预报）和时间编码决定，提供波形的宏观趋势。
*   **站点指纹 (Site Fingerprint/Steady-state)**：由站点固有的物理特性（效率、倾角、遮挡）决定，通过 **Site Latent Codebook** 进行解耦提取。

### 2. 模型架构组件 (Core Components)

#### A. 站点记忆库 (Site Memory Bank / Codebook)
*   **逻辑**：包含一组可学习的“典型站点原型”向量。
*   **少样本适配**：利用 1 个月的数据（探测信号）通过 **Attention 机制** 动态提取出目标站点的加权物理指纹。
*   **作用**：在预训练中沉淀多站点的物理共性，在适配期实现对新站点的快速迁移。

#### B. 扩散生成器 (Diffusion-based Generator)
*   **架构**：基于 **1D Conditional ResNet** 的扩散模型（DDPM）。
*   **FiLM 调制层 (Feature-wise Linear Modulation)**：受 *Consistent Light Transport (CLT)* 启发，将站点指纹和天气背景作为调制因子注入网络。
    *   **物理意义**：模拟物理上的“反射率/转换率”如何调节入射能量。它允许模型在保持整体能量守恒的同时，生成具有随机性但物理合理的“崎岖”细节。
*   **高效推理**：优化为 **1天 (96个时间点)** 的生成窗口，仅需输入目标日的天气预报和晴空 GHI 即可生成。

#### C. CPT 物理约束损失函数 (Consistent Power Transport Loss)
本框架舍弃了传统的平滑惩罚，采用了一种**“反平滑 (Anti-smoothing)”**的物理逻辑：
1.  **物理天花板 (ReLU Ceiling)**：利用 `pvlib` 计算的晴空 GHI 作为绝对上限，惩罚任何超限生成。
2.  **波动率匹配 (Fluctuation Matching)**：强制生成曲线的“崎岖程度（梯度绝对值）”在统计上与真实数据一致，确保生成数据具备真实云层遮挡产生的随机波动。
3.  **单调趋势约束 (Trend Consistency)**：惩罚发电量随日照强度增加而显著下降的情况（除非有气象异常），确保物理趋势的单调性。

### 3. 核心亮点 (Technical Highlights)

| 特性 | 描述 | 价值 |
| :--- | :--- | :--- |
| **物理锚点注入** | 通过 `pvlib` 将地理位置和时间转化为晴空 GHI 锚点。 | 确保模型具备基本的地理物理常识。 |
| **FiLM 调制机制** | 取代传统的特征拼接，实现更精细的特征耦合。 | 增强了模型对站点微小物理差异的表达能力。 |
| **反平滑优化** | 专注于波动率分布对齐，而非强制曲线平滑。 | 解决了生成数据因过度物理约束而导致的“太假/太滑”问题。 |
| **少样本泛化** | 冻结生成器，仅需 1 个月数据即可找准站点指纹。 | 极大降低了新站点并网初期的预测误差。 |

### 4. 数据流转 (Data Flow)
1.  **输入层**：[目标日 NWP (GHI, Temp)] + [晴空 GHI] + [时间编码] + [Site Latent]。
2.  **生成层**：Diffusion 从高斯噪声中通过 100 步迭代恢复波形，每一层受 FiLM 调制。
3.  **约束层**：通过 CPT Loss 实时修正波形的物理边界和波动风格。
4.  **输出层**：输出符合该站点指纹、带有真实崎岖细节的 96 点功率序列。

### 5. 适用场景
*   **新站并网**：仅有 1 个月历史数据时的预测能力建设。
*   **数据增强**：将 1 个月数据扩充为 12 个月的合成数据，用于训练下游重型预测模型（如 TFT）。
*   **极端天气模拟**：通过调整 NWP 输入，生成该站在极端天气下的可能功率响应。
